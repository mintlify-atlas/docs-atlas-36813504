The KERN API has two public authentication endpoints. Neither requires an existing token. Both return a `LoginResponse` containing a JWT access token and the full user object.

---

## POST /register

Registers a new user account and returns a JWT access token immediately. No email verification step is required.

**Auth required:** No

### Request body

<ParamField body="username" type="string" required>
  Unique display name for the account. Must not already exist in the database.
</ParamField>

<ParamField body="email" type="string" required>
  Email address. Must be unique across all accounts.
</ParamField>

<ParamField body="password" type="string" required>
  Plain-text password. Stored as a SHA-256 hash — never in plain text.
</ParamField>

<ParamField body="full_name" type="string">
  User's full name. Optional.
</ParamField>

<ParamField body="phone" type="string">
  Phone number. Optional.
</ParamField>

### Response — 200

<ResponseField name="accessToken" type="string" required>
  Signed JWT token. Valid for 30 minutes. Use as `Authorization: Bearer <accessToken>`.
</ResponseField>

<ResponseField name="token_type" type="string" required>
  Always `"bearer"`.
</ResponseField>

<ResponseField name="user" type="object" required>
  <Expandable title="user properties">
    <ResponseField name="id" type="integer">
      Auto-incremented user ID.
    </ResponseField>
    <ResponseField name="username" type="string">
      The username that was registered.
    </ResponseField>
    <ResponseField name="email" type="string">
      The email address that was registered.
    </ResponseField>
    <ResponseField name="full_name" type="string">
      Full name, if provided.
    </ResponseField>
    <ResponseField name="phone" type="string">
      Phone number, if provided.
    </ResponseField>
    <ResponseField name="avatar_url" type="string">
      URL of the user's avatar image. `null` on registration.
    </ResponseField>
    <ResponseField name="role" type="string">
      Always `"user"` for newly registered accounts.
    </ResponseField>
    <ResponseField name="bio" type="string">
      Short biography. `null` on registration.
    </ResponseField>
    <ResponseField name="has_completed_quiz" type="boolean">
      Whether the user has completed the onboarding questionnaire. Always `false` on registration.
    </ResponseField>
    <ResponseField name="assigned_routine" type="string">
      The routine type assigned after quiz completion. `null` on registration.
    </ResponseField>
    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp of account creation.
    </ResponseField>
    <ResponseField name="disabled" type="boolean">
      Whether the account is disabled. Always `false` on registration.
    </ResponseField>
  </Expandable>
</ResponseField>

### Errors

| Status | `detail` | Cause |
|---|---|---|
| `400` | `"El usuario o email ya existe"` | `username` or `email` is already taken |

### Example

<CodeGroup>

```bash curl
curl --request POST \
  --url https://kern-api.vercel.app/register \
  --header "Content-Type: application/json" \
  --data '{
    "username": "jaime",
    "email": "jaime@example.com",
    "password": "mypassword123",
    "full_name": "Jaime Gayo"
  }'
```

```json Response
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "user": {
    "id": 42,
    "username": "jaime",
    "email": "jaime@example.com",
    "full_name": "Jaime Gayo",
    "phone": null,
    "avatar_url": null,
    "role": "user",
    "bio": null,
    "has_completed_quiz": false,
    "assigned_routine": null,
    "created_at": "2026-05-14T10:00:00+00:00",
    "disabled": false
  }
}
```

</CodeGroup>

## POST /login

Authenticates an existing user with their email address and password, and returns a fresh JWT access token.

**Auth required:** No

### Request body

<ParamField body="email" type="string" required>
  The email address used when registering the account.
</ParamField>

<ParamField body="password" type="string" required>
  The account password in plain text. The API compares its SHA-256 hash against the stored hash.
</ParamField>

### Response — 200

Same `LoginResponse` structure as `POST /register`. See the response fields above.

### Errors

| Status | `detail` | Cause |
|---|---|---|
| `401` | `"Email o contraseña incorrectos"` | No account found for that email, or password hash does not match |

### Example

<CodeGroup>

```bash curl
curl --request POST \
  --url https://kern-api.vercel.app/login \
  --header "Content-Type: application/json" \
  --data '{
    "email": "jaime@example.com",
    "password": "mypassword123"
  }'
```

```json Response
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "user": {
    "id": 42,
    "username": "jaime",
    "email": "jaime@example.com",
    "full_name": "Jaime Gayo",
    "phone": null,
    "avatar_url": "https://res.cloudinary.com/demo/image/upload/sample.jpg",
    "role": "user",
    "bio": null,
    "has_completed_quiz": true,
    "assigned_routine": "Full Body",
    "created_at": "2026-05-14T10:00:00+00:00",
    "disabled": false
  }
}
```

</CodeGroup>
